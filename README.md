# Assistive Web Bot（无障碍浏览机器人）

一个面向无障碍使用场景的公益浏览器插件原型，支持：

- 网页化控制台：像网站一样操作自动采集与比价
- 一次输入关键词，自动在淘宝和京东抓取商品信息做比价
- 自动滚动页面
- 自动点击翻页元素（如“下一页”）
- 按 CSS 选择器采集页面数据
- 导出采集结果为 JSON
- 可选语音播报（启动/翻页/停止）

## 快速安装（Chrome/Edge）

1. 打开 `chrome://extensions/`（或 Edge 的 `edge://extensions/`）
2. 开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本项目目录：`/home/dreamking/browser-machine`
5. 点击扩展图标，选择「打开网页控制台」

## 使用方法

### A. 网页控制台（推荐）

1. 打开扩展弹窗，点击 `打开网页控制台`
2. 在控制台输入 `关键词`
3. 设置 `每个平台页数`
4. 点击 `启动多平台采集`
5. 任务运行后点击 `刷新结果` 查看统一表格
6. 点击 `导出 JSON` 进行后续比价分析

采集字段（默认）：

- `platform`：平台（taobao/jd）
- `keyword`：关键词
- `title`：商品标题
- `price`：价格（文本提取）
- `shop`：店铺（京东优先）
- `link`：商品链接
- `capturedAt`：采集时间

### B. 弹窗一键比价模式

1. 在插件里填写 `比价关键词`
2. 设置 `每个平台抓取页数`
3. 点击 `一键比价启动`
4. 完成后点击 `导出 JSON`

### C. 普通模式（自定义规则）

1. 打开目标网页
2. 点击插件图标，填写配置
3. 点击「普通模式启动」
4. 运行过程中可点击「当前页停止」
5. 采集完成后点击「导出 JSON」

## 注意事项

- 这是“网页化控制台 + 扩展采集端”的方案，不是纯公网网页采集。
- 纯网页会受同源策略/CORS 限制，无法稳定直接抓淘宝/京东。
- 淘宝/京东页面结构会变化，选择器可能需要定期维护。
- 部分页面（如 `chrome://`、扩展页）浏览器不允许注入脚本。
- 当前原型默认将数据保存在 `chrome.storage.local`。
